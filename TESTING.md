# Проверка NineCell

Этот чеклист помогает убедиться, что основные функции сервиса работают корректно.

## 1. Запуск через Docker

```powershell
cd C:\Users\marin\PycharmProjects\NineCell
docker compose up --build
```

Если папка проекта называется иначе, перейдите в фактическую папку проекта.

Открыть сайт:

```text
http://localhost:5000
```

RabbitMQ Management:

```text
http://localhost:15672
```

Данные RabbitMQ по умолчанию:

```text
ninecell / NCellRabbit-2026
```

## 2. Проверка входа

Демо-аккаунты:

```text
admin / NineCellAdmin-2026
user / NineCellUser-2026
```

## 3. Проверка задач

1. Войти как admin.
2. Создать задачу через кнопку `＋`.
3. Обновить страницу.
4. Задача должна остаться.
5. Переместить задачу в другую колонку.
6. Удалить задачу через красивое модальное подтверждение.

## 4. Проверка команд

1. Создать команду.
2. Добавить пользователя `user`.
3. Создать задачу внутри команды.
4. Войти во втором браузере как `user`.
5. Проверить, что командная задача видна.

## 5. Проверка личных задач

1. Войти как admin.
2. Создать задачу без команды и без исполнителя.
3. Войти как user.
4. Задача не должна быть видна обычному пользователю.

## 6. Проверка уведомлений

1. Назначить задачу пользователю.
2. Открыть панель “Уведомления”.
3. Проверить, что уведомление появилось.
4. Нажать “Очистить”.
5. Подтвердить действие в модальном окне.
6. Уведомления должны исчезнуть только у текущего пользователя.

## 7. Проверка Telegram

1. Указать в `.env` `TELEGRAM_BOT_TOKEN` и `TELEGRAM_BOT_USERNAME`.
2. Перезапустить контейнеры.
3. В Telegram нажать `🔐 Получить код`.
4. Войти на сайт через код.
5. Назначить задачу на этого пользователя.
6. Telegram должен отправить сообщение о назначении.

## 8. Проверка входящего потока

Во втором PowerShell:

```powershell
$body = @{
  source = "api"
  external_id = "demo-queue-001"
  title = "Срочно: баг оплаты"
  description = "Проверка RabbitMQ"
  tags = "bug, urgent, payment"
  team = "Victory Group"
  assignee = "user"
} | ConvertTo-Json

Invoke-RestMethod `
  -Uri "http://localhost:5000/api/incoming_tasks" `
  -Method POST `
  -Headers @{"X-API-Key"="dev-incoming-token"} `
  -ContentType "application/json" `
  -Body $body
```

Ожидаемый результат:

- API вернул `queued`;
- сообщение прошло через RabbitMQ;
- worker создал задачу;
- задача появилась на доске.

## 9. Проверка Retry

1. Открыть раздел “Очередь”.
2. Найти входящую запись.
3. Нажать `Retry`.
4. Подтвердить действие в модальном окне.
5. Запись должна повторно уйти в обработку.

## 10. Проверка System Health

1. Войти как admin.
2. Открыть раздел “Статус”.
3. Проверить PostgreSQL, RabbitMQ, Redis и счётчики очередей.

## 11. Проверка без Docker

```powershell
python -m venv .venv
.\.venv\Scriptsctivate
pip install -r requirements.txt
python app.py
```

Открыть:

```text
http://localhost:5000
```
